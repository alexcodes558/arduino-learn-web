Understanding Variables

üß† What a Variable Really Is

A variable is a named place in memory where Arduino stores information.

Think of memory as a row of boxes. A variable is a label on one of those boxes.

Instead of remembering a number, Arduino remembers the box name.

üì¶ Why Variables Exist

Without variables, you would have to hard-code every number.

Variables allow Arduino to:

  • Remember values
  • Change behavior
  • React to sensors

üî¢ Types Matter

Arduino needs to know what kind of data you are storing.

int ledPin = 13; float temperature = 22.5; bool buttonPressed = false;

Each type uses memory differently. Choosing the right one matters.

üîÅ Variables and loop()

Variables keep their value between loop cycles.

This is how Arduino remembers what happened before.

int count = 0; void loop() { count = count + 1; }

üö´ Common Beginner Mistake

Declaring variables inside loop() when they need to persist.

Variables declared inside a function can be reset each cycle.
