Module 1: How Arduino Code Really Works

What Is Arduino Code?

Arduino code is a simplified version of C++ designed to control hardware. Unlike apps or websites, Arduino programs talk directly to a microcontroller — a tiny computer whose only job is to read inputs and control outputs.

The Arduino Brain (Microcontroller)

When you upload code, it is stored inside the Arduino’s microcontroller. Once powered, the chip runs your code automatically — no keyboard, no screen, no mouse.

Program Flow: How Code Runs

Arduino programs follow a strict structure. There are only two required functions:

void setup() { // runs once when the board powers on } void loop() { // runs forever, over and over }

setup()

The setup() function runs exactly ONE time. This is where you configure pins, start communication, and prepare hardware.

  • Set pin modes (INPUT / OUTPUT)
  • Start Serial communication
  • Initialize sensors

loop()

The loop() function runs forever. When it reaches the bottom, it instantly starts again from the top. This creates continuous behavior like blinking LEDs or reading sensors.

Why Arduino Has No main()

Arduino hides the traditional main() function to make learning easier. Internally, Arduino wraps your code in an infinite loop so beginners can focus on logic instead of system setup.

Comments (Very Important)

Comments explain what code does. The Arduino ignores them, but humans need them.

// This is a single-line comment /* This is a multi-line comment */

Common Beginner Mistakes

  • Forgetting semicolons
  • Writing code outside functions
  • Using pins without pinMode()
  • Expecting code to run once instead of looping

Knowledge Check

Question: Which function runs repeatedly?

  • A) setup()
  • B) loop()
  • C) pinMode()

Correct answer: B

Module 2: Variables & Data Types

Variables store information the Arduino can use. You can create int, float, bool, and char variables to represent numbers, states, and text.

int ledPin = 13; bool isOn = false; float voltage = 5.0; char letter = 'A';

Module 3: Digital Output

Control LEDs and other components using HIGH and LOW signals.

pinMode(13, OUTPUT); digitalWrite(13, HIGH);

Module 4: Timing & delay()

Control how fast code runs and how long LEDs or components stay on/off.

digitalWrite(13, HIGH); delay(1000); digitalWrite(13, LOW); delay(1000);
